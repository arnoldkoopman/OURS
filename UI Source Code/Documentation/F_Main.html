Description

Mainform for the application.

The main dialogue has 3 different regions:
- Button row on top of the dialogue
- Page control in the middle of the dialogue
- Page control on the bottom of the dialogue.

The top button row is only available when started in test-mode. It gives the user control over the program flow and enables control of execution for each calculation step:
- [Open] enables the user to open a XML-file
- [Calculate] calculates the full project
- [Save] stores the results in the output XML-file and TXT-file for detailed results.

The page control in the middle is only visible in test-mode and shows all input data, itermediate results and final results:
- "Input" displays the input XML-file.
- "Project" displays the extracted project information and the calculation settings.
- "Receptors" displays the receptor points loaded from the XML-file.
- "Tracks" displays the tracks, track parts and trains loaded from the XML-file.
- "Sources" displays for each receptor point the calculated point sources, including the relevant measurement data from the database.
- "Ground" displays the retrieved ground data.
- "FEM" displays the results of the FEM-calculations for each ground scenario.
- "Postprocessing" displays the results of the post-processed FEM-calculations.
- "Uncertainty" displays the results of uncertainty of ground data and FEM-calculations.
- "Building" displays the results from the Building module.
- "Results" displays the final results calculated by the main formula.
- "Output" displays the output XML-file.

The bottom page control has 2 purposes. The first page ("Messages") is always visible (except in silent-mode) and gives information on progress, errors and warnings for the user. This information is always included in the XML output.
The other pages, only visible in test-mode, show the contents of the database distributed with the software ("OURS.sqlite" located in the program folder).

For command line options see OursMessage.TOursMessageList.ShowUsage
